# CloudSC DaCe CPU Pipeline

This directory contains a self-contained pipeline for optimizing, generating, and compiling the CloudSC microphysics scheme using DaCe, modeled after the `velocity` optimization stages.

## Structure

- `cloudsc_cpu_pipeline.py`: The orchestration script. It simplifies the SDFG, generates C++ code, flattens the build into `codegen/`, and auto-generates `cloudsc_main.cpp`.
- `cloudsc_main.cpp`: A C++ driver auto-generated by parsing the DaCe header. It handles ~160 arguments, allocates memory, and initializes physical constants.
- `cloudsc_python_ref_run.py`: Runner for the **Reference Python** implementation (direct NumPy).
- `compare_outputs.py`: Numerical validation tool using **Polars** and **SNR**.
- `codegen/`: Contains the flattened C++ source and headers extracted from the DaCe cache.
- `recompile.sh`: A standalone shell script for rapid C++20 recompilation.
- `input.h5`: Physical constants and input arrays from the original dwarf used to initialize all drivers.
- `env.sh` / `env.fish`: Environment activation scripts supporting Bash, Zsh, and Fish.

## Usage

### 1. Prerequisites
Setup the local Python environment and install dependencies. You will also need `libhdf5` installed on your system.

```bash
python3 -m venv .venv
source env.sh
pip install numpy h5py git+https://github.com/spcl/dace.git@main
```

### 2. Run the Pipeline
Generate the code and build the C++ binary:
```bash
# Release Build (C++20, -O3, -DNDEBUG)
python cloudsc_cpu_pipeline.py --release && ./recompile.sh
```

### 3. Execute & Serialize
You can run any of the three implementations. Use the `--save` flag to serialize all output arrays to HDF5 files for later comparison. Results are stored in dedicated directories with distinct prefixes.

**C++ Optimized Binary:**
```bash
./cloudsc_cpu_bin <steps> [--save]
# Example: ./cloudsc_cpu_bin 5 --save 
# -> Produces outputs_cpp/cpp_output_step_0.h5 ...
```

**Reference Python (NumPy):**
```bash
python cloudsc_python_ref_run.py --steps 5 --save
# -> Produces outputs_ref/ref_output_step_0.h5 ...
```

### 4. Compare Results
Use the comparison tool to validate the C++ binary against the reference Python implementation:
```bash
# Compare a specific step
python compare_outputs.py --step 0 --tol 1e-12
```
This script uses **Polars** to generate a summary table including Max Relative Error and **SNR (dB)**.


## How it works

1. **Optimization**: `cloudsc_cpu_pipeline.py` loads the SDFG and applies standard simplifications.
2. **Physical Constants**: The script reads `input.h5` (extracted from the original ECMWF dwarf) to populate over 100 physical constants and dimensions into the C++ driver.
3. **Codegen & Flattening**: DaCe generates raw C++ code. The pipeline "flattens" this by pulling the necessary source files into a local `codegen/` folder.
4. **Driver Generation (HDF5 Integration)**: 
    - The pipeline auto-generates `cloudsc_main.cpp` with a built-in HDF5 loader/saver.
    - At runtime, the binary loads input arrays (`PT`, `PQ`, etc.) from `input.h5`.
    - If `--save` is passed, it serializes all output fields after every step to `outputs_cpp/` using a shared HDF5 format.
5. **Multi-step Loop**: All drivers include high-resolution timers to measure average performance over multiple steps.
6. **Standalone Build**: `recompile.sh` is generated with detected HDF5 flags to allow C++ iterations without re-running the full DaCe flow.
